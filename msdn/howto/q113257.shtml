Как создавать новые документы без CWinApp::OnFileNew

Иногда необходимо создать окно, основанное на CMultiDocTemplate (другими словами, комбинацию CFrame/CDocument/CView) без использования CWinApp::OnFileNew.

Например, если программа имеет несколько шаблонов документов, CWinApp::OnFileNew будет спрашивать пользователя (в диалоговом окне) какого типа документ открывать. Программист может уже заранее знать, какого типа CMultiDocTemplate должен быть использован и, поэтому необходимость спрашивать пользователя отпадает.

Предположим, что приложение изначально создано при помощи AppWizard. Тогда для создания окна, основанного на CMultiDocTemplate, можно использовать недокументированную функцию CMultiDocTemplate::OpenDocumentFile. Для этого рассмотрим несколько шагов:

  1. Добавьте указатель CMultiDocTemplate в Ваш класс, наследованный от CWinApp:
          class CMyApp : public CWinApp
      {
        ...

       public:
        CMultiDocTemplate* m_pDocTemplate;

        ...

      }

    ЗАМЕЧАНИЕ: Если Вы планируете использовать несколько типов документов, то должны создать переменную-указатель CMultiDocTemplate для каждого типа документа.

  2. В функции CWinApp::InitInstance, уберите создание CMultiDocTemplate от начала и до AddDocTemplate. Установите указатель на новый CMultiDocTemplate. Используйте указатель для вызова AddDocTemplate:
          BOOL CMyApp::InitInstance()
      {
        ...

        m_pDocTemplate = new CMultiDocTemplate(IDR_TEXTTYPE,
                                      RUNTIME_CLASS(CMyDoc),
                                      RUNTIME_CLASS(CMDIChildWnd),
                                      RUNTIME_CLASS(CMyView));

        AddDocTemplate(m_pDocTemplate);

        ...

      }



  3. Для создания нового окна используйте метод указателя CMultiDocTemplate::OpenDocumentFile с параметром NULL. Для данного примера подразумевается, что в окне CView присутствует кнопка. В обработчике кнопки BN_CLICKED мы хотим создать окно, основанное на m_pDocTemplate:
          void CMyView::OnNewWindowButtonClicked()
      {
          CMyApp* pApp = (CMyApp*)AfxGetApp();
          pApp->m_pDocTemplate->OpenDocumentFile(NULL);
      }

    Эта же самая технология может быть использована и для окна, основанного на CSingleDocTemplate в однодокументном приложении (Single Document Interface (SDI)). Однако, в этом нет необходимости, так как такое приложение имеет всего один шаблон документа и, при вызове OnFileNew() создастся новое окно, не предлагая пользователю выбирать тип документа.

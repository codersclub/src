Куда движется Delphi

Введение

Данный документ - выдержки из отчета о BorCon2000. Источник http://www.xapxone.com/html/borcon2000_report.htm. Переведены куски, отнсящиеся к новым возможностям Delphi. Заранее приношу извинения за не очень качественный перевод.
Более подробно и другие обзоры смотрите на сайте Masters of Delphi

Новые возможности

Новые версии Delphi/C ++ Builder выглядят перспективными.
Code Insight дополнен новыми возможностями.Автоматическое завершение кода работает теперь внутри декларации класса, и будет иметь список перебора изменяемого размера. Runtime library также подверглась изменениям. Этодолжно помочь Kylix, runtime library - D6 и Kylix - будут во многом совместимы.
Все декларации variant и функции help вынесены в отдельный модуль, подключаемый при необходимости. Borland собираются обеспечивать "Microsoft-free" variant библиотеку, чтобы быть совместимыми с Linux.
Появился ряд новых unit: StrUtils, DateUtils, и ConvUtils. Данные модули будут содержать новые функции работы со строками и датами.
Изменения в IDE. Инспектор Объектов выделяет св-ва, которые являются просто указателями на другие компоненты. Он отображает св-ва и события для этих компонентов как под-компоненты родительского компонента.
Другая новость, что Winshoes будет включен в D6 как стандартный набор компонентов Интернета. Это - хорошая новость. Winshoes - широкий, мощный и открытый набор инструментальных средств Интернета, которые обеспечат большое поддержку, и на клиенте и на сервере.
Много стандартных действий, добавлено к Action List. Появятся так называемые ActionBars, упрощающие разработку меню, панели инструментов, actions. Они могут быть настроены во временя выполнения.
Добавлены средства тестирования ISAPI и CGI приложений. Каждое такое приложение сможет содержать более одного Web и DataModule. И наконец самое интересное, Pascal-евский язык написания скриптов. Возможно скоро это приведет к созданию "Delhi Server Pages".
Были представлены улучшения XML технологии, включая способность переместить данные от XML в TCLIENTDATASETS и наоборот.

так =).
теперь про анонимность http-proxy.
он не меняет твой IP.
он перенаправляет твои запросы через себя.

поэтому отвечающий веб сервер в поле заголовка
окружения скрипта имеет поле
HTTP-REMOTE-ADDR - равное ip анонимного прокси.

есть еще неанонимные http-proxy.
или просто - кэширующие.

они в заголовок http запроса подставляют поле
HTTP-X-FORWARDED-FOR или HTTO-CLIENT-IP или
кто на что горазд. но онсновные поля - эти.
и в них - значение твоего ip реального.

HTTP пере переменной означает, что эта
переменная взята из http заголовка запроса.

к примеру если ты обратишся к серверному
скрипту:

telnet server 80
GET http://server/cgi-bin/cgi.pl HTTP/1.0\n
f1: 123\n
f2: 234\n\n

у скрипта cgi.pl будут
следующие переменные :
HTTP_F1 = 123
HTTP_F2 = 234.

так же в перенаправляемый заголовок неанонимный
http-прокси подставляет
x-forwarded-for: ip_addr.

вот примерно так работают http-proxy сервера.

теперь по поводу www.anonymizer.com.
насолько помнится - он предоставляе анонимность
только платно.
а так - он работает просто как кеширующий прокси.

существуют целые команды, которые собирают
анонимные squid-ы, socks-proxy, shell-ы.
кто ищет - тот при упорстве их может найти =).

на данный момент могу дать пару быстрых анонимных squids:

с помощью которого тебя могут идентифицировать.

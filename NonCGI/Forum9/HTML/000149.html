Prefix: C = control (base+2)
D = data (base+0)
S = status (base+1)
Suffix: '-' for data inverted,
'+' data not inverted

Bit Pin Signal
C5- --- direction bit for bi-directional ports
C4- --- IRQ enable
C3- 17 -Select
C2+ 16 -Init
C1- 14 -AutoFd
C0- 1 -Strobe

S7- 11 +Busy
S6+ 10 -Ack
S5+ 12 +PaperEnd
S4+ 13 +SelectIn
S3+ 15 -Error

D7..D0 9..2 Data bits

Запись в порт данных напрямую задействует контакты(2..9) порта. Чтение с этого порта возвращает текущее состояние контактов. В архитектуре персонального компьютера используется такой принцип вывода, при котором каждый контакт адаптера состоит из двух транзисторов, причем только один из них является активным в определенный момент времени. Когда на контакт посылается бит "1", "старший" транзистор активизируется, т.е. "старший" транзистор замыкает контакт для прохождения тока, что символизирует логическую единицу.

Когда посылается бит "0", активизируется "младший" транзистор и размыкает контакты, что на этот раз означает логический нуль.

Контакты, имеющие "старший" транзистор, не могут быть соединены с другими выходами, следовательно, они не могут читать данные с внешних устройств. Тем не менее, порт данных может считать сигна лы, переданные на эти контакты. Это может быть полезно для целей диагностики, таких как проверка на соответствие битов. Если прочитанное значение не совпадает с последним записанным, то произошла ошибка.

Современные параллельные порты могут отключить оба транзистора, позволяя контакту "плавать" и получать данные с внешних устройств. Эти контакты имеют три состояния и могут выполнять двунаправленный ввод-вывод.

Порт состояния - это порт только для чтения. Он возвращает информацию о значении битов 3-7 считанного байта (где бит 0 является наименьшим значащим битом). Отметим, что бит 7 - инвертированный, т. %. когда контакт замкнут, значение бита равно нулю.

Порт управления - порт обмена информацией с битами 0..3, соответствующими контактам ввода-вывода. В отличие от порта данных, этот порт задействует контакты только с одним транзистором, напряжение которого соответствует логическому нулю. Когда программа посылает "0" в какой-либо бит порта управления, транзистор размыкает соответствующий младший разъем. Если биту присваивается зна чение "1", транзистор перестает быть активным и по существу ведет себя так, как будто связь с ним отсутствует. Как и в приведенном выше примере о контактах с тремя состояниями, этот контакт можно назвать "плавающим ". Такая конфигурация называется open collector output.

Подобно контактам с тремя состояниями, open collector output допускает двунаправленный ввод-вывод, однако, имеет несколько отличий, которые особенно полезны при работе с устройствами ввода-вывода. Помимо чтения информации с контактов, порт управления выполняет еще несколько функций. Так, бит 4 используется для включения запроса прерывания (IRQ) от внешнего устройства. Если он установлен в момент, когда контакт S6 перемещается к "старшему" транзистору, произойдет аппаратное прерывание. Бит 5 не используется в стандартных параллельных портах, но он отвечает за направление движения битов в современных двунаправленных портах данных. Когда этот бит установлен, оба транзистора на линиях данных отключены, что позволяет внешнему источнику управлять контактом.

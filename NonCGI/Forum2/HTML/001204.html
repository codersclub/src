Автор Тема:   Проблема с SendMessageCallback
zAg35 опубликован 22-09-2001 17:38 MSK        

Поток следит за событием, а когда то проиходит отправлеет сообщение в Form2 :

void __fastcall TMyThread::Execute()
{
MSG msg;
//создаем очередь потока как рекомендует MSDN
PeekMessage(&msg, NULL, WM_USER, WM_USER, PM_NOREMOVE);

while (!Terminated){
//ждем событие
WaitForSingleObject(hEvent,INFINITE);
//сбрасываем
ResetEvent(hEvent);
SendMessageCallback(Form2->Handle,32773,0,0,(SENDASYNCPROC)&EchoProc,AppData);
//GetMessage необходим чтобы вызвалась EchoProc
GetMessage(&msg,NULL,0,0);
}
}

//функция отклика
VOID CALLBACK EchoProc(HWND hwnd,UINT uMsg,ULONG_PTR dwData,LRESULT lResult)
{
//Что нибудь делаем
.......

}

Так вот , проблема в следующем:
Первый раз все работает нормально - когда hEvent устанавливается в сигнальное состояние поток просыпается, SendMessageCallBack вызывает оконную функцию Form2 ,возвращаемся из нее попадаем в EchoProc а дальше все -
поток больше не реагирует на изменения hEvent.

При этом если использовать SendMessage то все работает нормально.
Вы можете спросить а на кой хрен использовать SendMessageCallback+GetMessage , если можно использовать
один SendMessage без этого изврата - просто интересно почему так проиходит.

Есть соображения по этому поводу ?

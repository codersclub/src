Автор Тема:   Builder C++. DefineProperties не работает.
sps опубликован 09-11-2001 19:13 MSK        
Вопрос: почему выбрасывается сообщение
"Error reading COMPONENT->PROPERTY.PROPERTY: Invalid property path. ...".
Описание: есть следующая ситема классов.
class A
{
AnsiString m_AS;
TPicture *m_pP;
public:
__property AnsiString AS = {read=..., write...};
__property TPicture *PP = {read=..., write...};
};
class B : public TPersistent
{
typedef map<int, A> TMA;
TMA *m_pMA;
protected:
void __fastcall MAWriterProc(TWriter* pW)
{
}
void __fastcall MAReaderProc(TReader* pR)
{
}
void __fastcall DefineProperties(TFiler* pF)
{
TPersistent::DefineProperties(pF);
pF->DefineProperty("PMA", MAReaderProc, MAWriterProc, true);
}
public:
__property PMA = {read=..., write=...};
};
class C : public TCustomComponent
{
B *m_pB;
__published:
__property *PB = {read=..., write=...};
};
Т.е. в данном случае сообщение следующее:
"Error reading C->PB.PMA: Invalid property path. ...". На самом деле функция MAWriteProc() работает нормально, т.е. при просмотре формы как текст видны сохраненные данные. А вот при попытке переключиться обратно в режим просмотра формы выдается выше указанное сообщение. Причем, в функцию MAReadProc() даже не попадаем - ни до сообщения, ни после.

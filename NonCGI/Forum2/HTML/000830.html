Во-первых обновления в окнах может и не быть по сообщению WM_PAINT (а например по WM_NCPAINT, или непосредственно по получению хэндла DC из потока. вариантов уйма).
Во вторых я не могу получить то, что находится непосредственно под моим окном.
В третьих высылаю упрощеную программу. посмотрите, может что не так делаю.
1. Создайте обычное MFC диалоговое приложение.
2. Добавте переменную CPoint m_ptShadow;
3. Добавте в OnInitDialog строки:
CRect rcWnd;
GetWindowRect(rcWnd);
m_ptShadow.x=rcWnd.left+rcWnd.Width();
m_ptShadow.y=rcWnd.top;
4. Перехватите WM_MOVE:
void CTest123Dlg::OnMove(int x, int y)
{
CDialog::OnMove(x, y);
// TODO: Add your message handler code here
CRect rcWnd;
GetWindowRect(rcWnd);
::RedrawWindow(::WindowFromPoint(m_ptShadow),NULL,NULL,RDW_INVALIDATE | RDW_UPDATENOW | RDW_ERASE );
HDC hDC=::GetDC(NULL);
for(int y1=0;y1<10;y1++)
{
for(int x1=0;x1<10;x1++)
{
COLORREF col=::GetPixel(hDC,rcWnd.left+rcWnd.Width()+x1,rcWnd.top+y1);
::SetPixel(hDC,rcWnd.left+rcWnd.Width()+x1,rcWnd.top+y1,RGB(GetRValue(col)/2,GetGValue(col)/2,GetBValue(col)/2));
}
}
::ReleaseDC(NULL,hDC);
}
p.s. Не смотрите на неэффективность. Лучше посмотрите какие глюки получаются.
И в чем я ошибаюсь?

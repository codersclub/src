Разобрать .exe в исходник реально!!!
===========================================
I.
Вначале нужно установить на каком языке написана прога: для этого ищем в файле такие типы строчек: "Borland", "Microsoft", "Visual", "Symantec", "Delphi" итп. EXE может быть запакован-надо распаковать. ASM-исключение :))

II.
Теперь нужно выяснить для какой платформы писалась программа: тут надо смотреть EXE заголовки (HIEW, NC..). Если есть такие слова, типа: ".text", ".data", то это программа для WIN32(Windows9X). При этом еще должна быть "PE"-сигнатура. Если есть "NE"-сигнатура, значит ето Win 3.11... И последний случай - программа под DOS(удовл. пункту I и нет PE/NE).

III.
Дизассемблирование: if(WIN32) берем W32DASM/TD32 else if(DOS) берем CV/TD. Win311 уже давно похоронен как самый уродский тип EXE ;)

IV.
Получение main/WinMain. Тот начальный код, который выдал дизасм не есть истинная программа!!! Вначале всех прог, написанных на языках высокого уровня идет функция инициаализации, а в конце нее происходит вызов на main/WinMain. Конкретные исходники функции инициализации можно найти в компиляторах(напр. crt0.c у VC++, или \SRC\*.c у Borland C++ 3.1). Короче точка входа в main везде разная. Ищите и найдете...

V.
Перевод ассемблерных кодов в команды языка.
На примере с C/C++:
Вызов функции:
ASM: C/C++:
push 0 func(0,1,2), или
push 1 func(2,1,0)
push 2 //см. CALLING CONVENTION
call 0123
структура if:
mov ax, var if(var)
test ax, ax
je label {
... ...
label: }

СОВЕТ: попробуйте посмотреть свою прогу и все поймете. ДИЗАСМИНГ - это круто!!!
============================================

Я вот посмотрел, и возникло сразу несколько вопросов.
1) какой смысл дублироваь вызовы для полчуния адреса и IP компа типа getgostbyname асинхронными аналогами WSAGetHostByName если у него даже не стоит обработчик сообщений окну, посылаемых этой асинхронной функцией.
2)почему работает подобное приведение
(LPSTR)(LPCTSTR)m_cshostname
ведь мы приводим указатель на константу к просто указателю, вроде должна выдаваться ошибка на этапе компиляции
3)Функции ProcessPendingAccept производиться принятие входящего соединения. В ней создаеться сокет,которые будет соединен с клиентским сокетом.
CClientSocket* pSocket = new CClientSocket(this);

if (m_MySocket.Accept(*pSocket))

но потом этот pSocket никак не сохраняеться.
Но и никак не унитожаеться. Вроде бы такой сокет после выхода из функции дожен уничтожаться, как вышедший из области видимости.
Даже в МСДН так написано.
Note The Accept member function takes a reference to a new,
empty CSocket object as its parameter. You must construct this
object before you call Accept. Keep in mind that if this socket
object goes out of scope, the connection closes.
Но этот сокет релаьно все же продолжает существовать и принимать сообщения.

Может дело в том, что этот сокет создан оператором new. Вот тчо написано в МСДН
If you created the socket object on the stack, its destructor is
called when the containing function goes out of scope. If you created
the socket object on the heap, using the new operator, you are
responsible for using the delete operator to destroy the object.
Но разве в VC не работает автоматичесикй "сборщик мусора" который дожне удалять все обьекты созданные при помощи new, котрые вышли из области видимости.
Хотя может я чего-то не понимаю.
4) Не относиться к проге, просто интерестно. Как API-шными функциями узнать, что сокет на другом конце закрыл соединение и нужно ли после того, как он закрыл соединение вызывать closesocket?

Заключение;) в проге конечно много ошибок, может поэтому исполняемые файлы скомпилины в Debug, но как ни страннно, все это работает. Так что прога полезная даже в этом смысле - разобраться, почему вроде как код с ошибкой - а работает.

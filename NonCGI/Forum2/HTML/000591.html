и видим там два файлика с названиями:

Diasrv32.zip
Первый - клиент, второй - сервер.
Для lamo больше подойдёт второй :)
протокол обмена пакетами UDP полность заимствован у BO2K.
файлы DSocket.cpp и DSocket.h не нужны, я их просто забыл от туда убрать :)
файлы DString.cpp и DString.h нужны - в них хранится обгрызанный MFC класс CString :) просто меня в то время очень прикалывало копаться в кишках MFC.
В тоже время никакого MFC здесь не присутствует - это видно по отсутствию всяких там файликов типа stdafx и т.д.
Изначально проект собран в VC++ 4.2, но я так понимаю, что собрать в другом компиляторе не составит никакого труда, т.к, всё, что нужно - это подключить в проект DString.cpp, Main.cpp и Wsock32.lib. А в клиенте ещё не забыть Dia.rc.

В данной версии, клиент определяет имя пользователя, под которым был осуществлён вход в сети мелкософта и отправляет запрос серверу. У сервера в функции WinMain есть такая строчечка:
AccessUsers = "AdministratorOlkaAnna";
Сервер просто сравнивает в пришедшем пакете имя пользователя с содержимым данной строчечки ну и выдаёт соответствующий ответ :) Поэтому от ответа зависит - появится ли у клиента аськовая ромашка в трее или нет.

Кстати у клиента всё в той же функции WinMain регистрируется горячая кнопочка Alt-E при нажатии которой ромашка скрывается от глаз начальника :)

Передаваемые пакеты упаковываются в структурку типа DIA_HEADER и на стороне приёмника соотвественно распаковываются в подобную структурку.

Код я постарался прокомментрировать по максимуму, так что если всё же будет чего не понятно, то готов выслушать вопросы.

итак, lamo, нам от тебя нужен айпишник, на котором будет сидеть программка, которая будет ожидать UDP пакеты, например, на 5000-ом порту.
Допустим приходящие пакеты будут формата:
struct {
int cod; //номер команды
int port; //порт клиента, на котором он ожидает входящих UDP пакетов
int lenghtName; //длина имени клиента
char name; //само имя клиента
}

допустим, что cod равняется 1 (что к примеру соотвествует выходу пользователя в онлайн)

Например у этой программки будет сидеть внутри строчечка:
char "lamoGLDEiLwebmaster";

соотвественно программка сравнивает name из поступившего пакета с этой строчечкой, и, если name содержится в строчечке, то в имеющийся в программке массив типа:


char mass {'1084','194.52.3.56','lamo',
'1293','213.54.182.11','GL',
.......
}
добавляется соответствующая строчечка,
а по обратному адресу пришедшего пакета отправляется следующий пакет:

struct {
int cod; //допустим 2
int lenghtMass; //длина ентого массива
char *mass; //собственно сам массив

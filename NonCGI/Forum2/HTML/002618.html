Долгое объяснение:
вот собственно в чем проблема - есть библиотека, которая грузится динамически (она в виде dll). Библиотека реализована на gnu C по windows. Нужно из этой библиотеки настоить callback в мой элемент ActiveX. настроить callback получается только, если функция адрес которой передается в dll объявлена как static и данные которыми она оперирует объявлены анологично. Но такая функция и данные становятся общими для всех ActiveX, загруженных в этот контейнер. Вот тут то и проблема - мне нужно, чтобы дынные ни в коем случае не пересекались.
Поэтому нужно, чтбы каждый раз заново создавался экземпляр объекта по полной программе - т.е. статические переменные создавались тоже заново

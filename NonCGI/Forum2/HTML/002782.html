Попробуй

SetWindowPos
The SetWindowPos function changes the size, position, and Z order of a child, pop-up, or top-level window. Child, pop-up, and top-level windows are ordered according to their appearance on the screen. The topmost window receives the highest rank and is the first window in the Z order.

BOOL SetWindowPos(
HWND hWnd, // handle to window
HWND hWndInsertAfter, // placement-order handle
int X, // horizontal position
int Y, // vertical position
int cx, // width
int cy, // height
UINT uFlags // window-positioning options
);

Сейчас открыл Джеффри Рихтера "Создание Win32 приложений". Итак.
Он описывает перехват API вызовов. Т.е. можно перехватить любые обращения к SetWindowPos() из user32.dll. Т.е. ты отслеживаешь обращение к SetWindowPos() или к SetWindowPlacement() из всех процессов системы. Затем внутри твоей функции перехвата определяешь, что вынести наверх хотят твое окно, и не разрешаешь ( обрываешь вызов).
Там же ( у Рихтера ) есть примеры. Если надо, могу выслать ( но в воскресенье ).

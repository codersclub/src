Запуск Java-программы. Пути. Переменные PATH и CLASSPATH. (Angie)
Средствами стандарного JDK:

Несмотря на то, что набор JDK предназначен для создания программ, работающих в графических средах (MS Windows, X Window System), он ориентирован на выполнение из командной строки окна MS-DOS Prompt в Windows 95/98/Me или окна Command Prompt в Windows NT/2000. В системах *nix можно работать и текстовом режиме и в окне Xterm. Написать программу на Java можно в любом текстовом редакторе (NotePad, WordPad или vi, emacs). Файл с программой сохраняем с расширением *.java.

Потом из командной строки вызывавается компилятор:

javac имя_файла.java

который создает в том же каталоге по одному файлу на каждый класс описанный в программе и расширением *.class.

Далее из командной строки вызывается интерпретатор:

java имя_файла

При этом системная переменная PATH должна содержать полный путь к подкаталогу bin каталога с JDK (например в Windows: PATH=C:\jdk\bin), а переменная окружения CLASSPATH должна указывать пути к архивным файлам и каталогам с библиотеками классов (например в Windows: CLASSPATH=С:\jdk\myprogs;C:\jdk\myarh.jar;.;).

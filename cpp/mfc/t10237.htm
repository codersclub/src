Hi!
The problem here is not with the way the ODBC is being used. It is the =
way the Access database does locking of its tables. Access goes with the =
locking of a Pagesize of 2K in a table. Thus it always locks more than =
one record giving rise to the conflict. One way to solve this: Pad each =
record with empty field(s) so that the size is 2K and Access will not =
start a new record in part of a page.=20

CStandardDialog and anything that doesn't process goes to the final

   Well in effect it's the other way around: anything that isn't handled in 
_your_ dialog class gets passed down to CDialog or the next highest class 
with a handler (a more complete view would note that calls first get bounced 
up from the original receiving class to the highest class which has a 
handler).

> Well, that doesn't seem to be happening. I've placed breakpoints in

   All I can think of is that you did use ClassWizard to defined these, 
right?  There should be declarations for these inside the AFX_MSG and 
AFX_MSG_MAP macros (in the header and source file respectively).

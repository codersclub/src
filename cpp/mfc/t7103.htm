displayed. I have overwritten OnGetExtent() for both Server items,

I'm not sure I understand your question. _How_ do you want both views
to be displayed? Do you want the client to offer your server two rectangles
in its window, one for each view?  Or do you want to split the rectangle
you're offered into two windows and use one for each of the views?

>CMyDoc::OnGetEmbeddedItem() gets called and I return
>the first Server item, but CMyDoc::OnGetLinkedItem(LPCTSTR lpszItemName)
>never gets called to return  my second Server item.

If you're being inserted as an embedded object, MFC will call
OnGetEmbeddedItem().  If you're being inserted as a linked object, MFC
will call OnGetLinkedItem().  If OnGetLinkedItem() isn't getting called,
it's only because you're getting embedded and not linked.

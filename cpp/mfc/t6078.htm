Hello,

Be aware that the MSComm control, does not allow speeds above 19200.  I was
looking into using the MSComm control too, until I realized that it could not
handle 57600.  I also ran into problems when trying to send data with embedded
NULLs (0x0).   I looked at the MicroHelp comm control, and it too, had problems
sending embedded 0x0's.  I currently use the Win32 api for communications.

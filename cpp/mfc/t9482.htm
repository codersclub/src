     I must admit it is a little out of the normal to remove doc/view 
     templates. Here is my reason for doing so :
     
     The application I am building allows subsets of the application to be 
     dynamically loaded. The subsets are defined in MFC extension DLL's. In 
     some cases the subsets register doc/view templates. When I shutdown 
     the application I unload the dynamically loaded subsets that were 
     loaded during the user session. Just prior to unloading the DLL's the 
     DLL is asked to clean up for shutdown. If the DLL has registered 
     Doc/View templates I would like to remove them..

>> I have observed the the first extension dll is loaded twice.
>> ( which seems strange )

Physically loaded in two different places?  What exactly do you mean
by "loaded twice"?  You will be "attached" to twice, since the DLL
and the EXE both attach to the DLL.

>> I can breakpoint the process of attaching  the first instance.
>> It looks like the main program uses the second instance.

App A links to DLL B
App A links to DLL C
DLL B links to DLL C

If dynamic references from DLL B are resolved first, then DLL B will
resolve it's references into DLL C before the application does.  This
would be what you're seeing.  It's not an error.

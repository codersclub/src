We have lot's of database access stuff (At the moment DAO but perhaps

several dlls. For example, to access accounts one would use the

You should read the technote which describes DLL programming. It outlines a 
way to use AfxLoadLibrary() instead of ::LoadLibrary(), and AfxFreeLibrary() 
instead of ::FreeLibrary(), and how to write a call to AfxTerminateLibrary() 
in your DllMain() function so that MFC can hook and unhook your library as 
you decide to load and discard it.

This will only work in MFC 4.0 or newer.

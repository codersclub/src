I have a MDI application (16-bit) that is composed of different
CFormViews.  Embedded within these views are single and multi-line edit
controls.  When displaying multiple views, if the user clicks on the
edit control of an inactive view, the view containing this control does
not become active.  Clicking on the frame of the window or another
control in the view (i.e. - listbox) makes the inactive view active.
Is there something special I need to do so that the edit control
behaves in a similar manner?

I have just dealt with a similar problem. The problem was caused by the edit
control having the WS_EX_NOPARENTNOTIFY style. With this style removed, the
view was activated properly.

This problem was only occuring for us when our 16-Bit application was running
under 95 (Windows 3.1 was OK).

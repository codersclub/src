On the server side, I typically call in order:
InitializeSecurityDescriptor()
SetSecurityDescriptorDacl() // needed to grant access
CreateNamedPipe()
ConnectNamedPipe()
...

On the client side,
InitializeSecurityDescriptor()
SetSecurityDescriptorDacl() // needed if you want the server to impersonate
WaitNamedPipe()
CreateFile()
...

Sounds like you are not setting the DACL.  Refer to Christopher Nefcy's
"Windows NT Security" (http://www.microsoft.com/win32dev/netwrk/ntprog.htm)
and others at http://www.microsoft.com/win32dev/netwrk/seccpp.htm.

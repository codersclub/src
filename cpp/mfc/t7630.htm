Modal CFrameWnd??? 

Roger M. -- RUNDEWOOD@cerner.com
Friday, November 08, 1996

Environment: Win 95, VC++ 4.0

I have the need to display a CFrameWnd (with a CDocument and CFormView)
modally from a regular DLL when an OLE automation method is called.  I
have tried creating a CDialog, calling DoModal on it, making it
invisible by calling MoveWindow(0,0,0,0) in its OnInitDialog, and then
immediately calling LoadFrame to create my CFrameWnd-derived object. 
The dialog is hidden fine, the frame window is created fine, but the
frame window even though parented to the dialog is not modal to the
calling application.

The question:  Am I on the right track?  Should I be trying to do this
some other way?  Any suggestions would be greatly appreciated.




Doug Brubacher -- Doug_Brubacher@compuware.com
Monday, November 11, 1996

     MFC displays Modal dialogs by simply disabling the parent.
     
     You can use this same trick by disabling your calling application when 
     displaying your CFrameWnd.  In that case you probably would not need 
     this modal dialog stuff.

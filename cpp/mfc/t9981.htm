Gosh.. I just did that.
It's is not difficult actually to do that once you understand the
CSplitterWnd's implementation.
First of all, derive a new class from CSplitterWnd.
Add your own WM_SETCURSOR, WM_LBUTTONDOWN & WM_MOUSEMOVE.
In all of the event handler above, merely call CWnd::OnXXXXX where XXXX is
the appropriate base member function.
eg
CMySplitterWnd::OnSetCursor(...)
{
CWnd::OnSetCursor();
}
that's it. you are set.

I believe this is the easiest yet the most efficient method.

Override the following three virtual functions and two messages

virtual void StartTracking(int ht);
virtual void StopTracking(BOOL bAccept);
virtual void SetSplitCursor(int ht);

WM_MOUSEMOVE and WM_SETCURSOR

implementation in your derived class will do nothing!

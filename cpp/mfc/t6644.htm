Environment: MSVC 4.1, Win95

I'm doing that. Using  document-view architecture, both the list
and the tree view are views on the same doc. A click on the 
tree causes it to do a doc->UpdateAllViews(HINT_RELOAD_LIST,filter).
Then when the listview gets the HINT_RELOAD_LIST with the associated
filter, it goes to the doc to get the information expected of it.

Doing the splitter view itself is just cookbook out of the docs or 
out of Programming Win95 with MFC.

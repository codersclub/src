-----From: Stuart Downing 

This comes up because MFC 4.2 introduces some new entry points to the =
template collection classes.

Here are two possible solutions to this problem:
1) If you are only using the collection for containment, and you don't =
care about serialization, change
the BASE_CLASS (CObList) template parameter to CPtrList.
2)  If you need serialization support, make sure the compiler knows that =
CMyObject is derived from=20
CObject before the declaration of CMyObjectList.  The following is not =
sufficient...
class CMyObject;
CTypedPtrList CMyObjectList;

Instead...
class CMyObject : public CObject
  {...};
CTypedPtrList CMyObjectList;

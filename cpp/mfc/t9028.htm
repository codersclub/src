Make sure the license file is available when the control is inserted.
If it isn't in the same directory as the control's executable, there
will be trouble.

You could figure out exactly what's going on by using the debugger.
Visual C++ comes with a very, very powerful debugger.  You could set
the executable image for your debugging session to be TSTCON32.EXE,
and put a breakpoint on the licensing member functions within  your 
control.  You'd be albe to trace through the code and see _exactly_
what fails right as it's happening!

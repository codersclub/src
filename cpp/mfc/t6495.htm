Problem linking static library into dynamic link library 

Chul Bae -- chulb@morgan.com
Wednesday, August 28, 1996

Environment: VC++ 1.52, Windows 3.11

We have created a static library which contains files using
IMPLEMENT_DYNAMIC() and MESSAGE_MAP macros.  When we try and link in this
static library into our dynamic link library we get the following
linking errors:

-error L2029: 'public: static struct CRuntimeClass __near
CObject::classCObject' : unresolved external
-protected: static struct AFX_MSGMAP __near CPropertySheet::messageMap' :
unresolved external

It appears as if the compiler is not properly expanding the macros.

Any suggestions?



Rajat Singhal -- rajat@corus.com
Saturday, August 31, 1996


Chul,

I think you haven't put the corresponding DECLARE_DYNAMIC() macro in the
public section of your class. I had a similar problem once. This solved
it.

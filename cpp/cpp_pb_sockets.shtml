Клиент и сервер с использованием сокетов.

Здесь представлено простейшее приложение, показывающее как отправлять различные команды, запускать файлы и передавай файлы обратно. Пример предназначен в основном для начинающих.

Сервер

 

В архиве содержится два приложения SockCommander (SC) и SockListener (SL). SL выступает в роли сервера и ожидает соединения на 777 порту. На SC указывается IP адрес и порт, нажимается кнопка соединения, далее можно вводить команды, нажимаю после этого на "Отправить".

На данный момент SockCommander умеет понимать следующие команды:

1.  CMD: <ваша команда>
2.  FILEGET:<путь файла, который мы хотим скачать>
3.  !SHUT

Описание команд:

1. CMD используется для посылки команд, которые могут быть выполнены сервером. Например команда CMD:Notepad.Exe заставит сервер запустить notepad
2. FILEGET используется для получения небольших файлов с сервера. Например FILEGET:c:\msdos.sys (не забудьте указать допустимый путь для сохранения файла в File Save)
3. !SHUT используется для шутдауна сервера
